> 说明：本文档用于临床学习、科室教学和科普写作，不替代本院诊疗规范、上级医师判断、专科会诊意见或患者个体化诊疗。

## 推荐结构

每个专题单独一个文件夹，尽量保持同样的文件命名：

```text
clinical-notes/
├── README.md
├── shoulder-dislocation/
│   ├── README.md
│   ├── xiaohongshu.md
│   └── references.md
├── templates/
│   ├── topic-template.md
│   ├── xiaohongshu-template.md
│   └── references-template.md
└── .gitignore
```

建议约定：

- `README.md`：专业版主文档，适合 GitHub 首页阅读。
- `xiaohongshu.md`：大众科普版，适合改写后发布到小红书。
- `references.md`：参考文献和指南链接。
- `images/`：如需要配图，放在专题目录下的 `images` 文件夹。
- `cases/`：如需要病例复盘，注意完全去标识化后再保存。

## 每篇专题建议包含

1. 一句话总结
2. 急诊评估流程
3. 危险信号和会诊指征
4. 检查与影像选择
5. 处理原则和操作要点
6. 出院医嘱和随访
7. 病历模板或复盘清单
8. 患者科普版本
9. 参考文献

## 证据标记

为了方便日后更新，可以在文中使用简单标记：

- `指南/共识`：来自指南、共识或专业学会建议。
- `系统综述/Meta`：来自系统综述或 Meta 分析。
- `RCT`：来自随机对照试验。
- `队列/病例系列`：来自观察性研究。
- `临床经验`：来自个人复盘、科室流程或专家经验，需要定期回看。

## 更新规则

- 重要专题建议至少每 6-12 个月复查一次文献。
- 遇到指南更新、药品说明书变化、本院流程调整，应及时修改。
- 每次更新建议在文末或提交记录中写清楚更新日期和主要变化。
- 高风险内容，如镇静、局麻剂量、抗菌药物、狂犬病暴露处置，应优先核对最新指南和本院制度。

## 发布建议

GitHub 适合放专业版知识库，小红书适合放删减后的患者科普版。

发布患者科普时建议：

- 不写可被普通人模仿的高风险操作步骤。
- 不展示患者隐私、影像号、住院号、面部、纹身等可识别信息。
- 不承诺疗效，不替代线下就诊。
- 对急症和危险信号写清“及时就医”。
